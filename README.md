# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Vite

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения (MVP \+ событийная модель)

Приложение построено по паттерну **MVP (Model‑View‑Presenter)** с использованием **событийно‑ориентированного подхода** через `EventEmitter`.

### Model (`Products`, `Basket`, `Buyer`, `WebLarekApi`)

Отвечает за данные и бизнес‑логику:

-   хранит и управляет данными;
    
-   реализует бизнес‑логику (добавление в корзину, валидация данных);
    
-   взаимодействует с API.
    

### View (`Header`, `Gallery`, `Modal`, `CardCatalog` и др.)

Отвечает за отображение:

-   отображает данные в DOM;
    
-   не содержит бизнес‑логики;
    
-   эмиттит события при взаимодействии пользователя (клик, ввод).
    

### Presenter (код в `main.ts`)

Координирующий компонент:

-   координирует взаимодействие Model и View;
    
-   подписывается на события от View и моделей;
    
-   вызывает методы моделей для изменения данных;
    
-   обновляет View при изменении данных.
    

## Ключевые классы и интерфейсы

### Базовый код (`components/base/`)

#### 1. `Component<T>` — базовый класс для всех UI‑компонентов

Базовый строительный блок всех визуальных компонентов.

-   конструктор: `constructor(container: HTMLElement)` — принимает контейнер для рендера;
    
-   поле: `container: HTMLElement` — ссылка на DOM‑элемент‑контейнер;
    
-   метод: `render(data?: Partial<T>): HTMLElement` — рендерит компонент с данными.
    

#### 2. `Api` — базовый класс для HTTP‑запросов

Обеспечивает взаимодействие с сервером.

-   конструктор: `constructor(baseUrl: string, options: RequestInit = {})` — инициализирует API с базовым URL;
    
-   методы: `get(uri: string)` — GET‑запрос, `post(uri: string, data: object)` — POST‑запрос.
    

#### 3. `EventEmitter` — брокер событий (паттерн «Наблюдатель»)

Механизм коммуникации между компонентами.

-   поля: `_events: Map<string | RegExp, Set<Function>>` — хранилище обработчиков событий;
    
-   методы:
    
    -   `on(event, callback)` — подписка на событие;
        
    -   `emit(event, data)` — генерация события с данными;
        
    -   `trigger(event, context)` — вызов обработчиков события.
        

### Модели данных (`components/models/`)

#### 1. `Products` — каталог товаров

Управляет данными о товарах.

-   `setItems(items: IProduct[])` — загружает товары в каталог;
    
-   `getItems(): IProduct[]` — возвращает все товары;
    
-   `getItem(id: string): IProduct` — ищет товар по ID (выбрасывает ошибку, если не найден);
    
-   `findItem(id: string): IProduct | null` — ищет товар, возвращает `null`, если не найден;
    
-   `setSelectedItem(id: string)` — устанавливает выбранный товар для детального просмотра;
    
-   `getSelectedItem(): IProduct | null` — возвращает выбранный товар.
    

#### 2. `Basket` — корзина покупок

Управляет корзиной пользователя.

-   `addItem(item: IProduct)` — добавляет товар в корзину;
    
-   `removeItem(id: string)` — удаляет товар по ID;
    
-   `clear()` — очищает корзину;
    
-   `getTotal(): number` — возвращает общую стоимость товаров в корзине;
    
-   `getCount(): number` — возвращает количество товаров в корзине.
    

#### 3. `Buyer` — данные покупателя

Хранит и валидирует данные заказчика.

-   геттеры/сеттеры для `payment`, `address`, `email`, `phone` — управление данными формы;
    
-   `validate(): BuyerErrors` — валидирует данные, возвращает объект с ошибками;
    
-   `clear()` — очищает данные после успешного заказа.
    

#### 4. `WebLarekApi` — слой коммуникации с сервером

Конкретная реализация API для проекта.

-   `getProducts(): Promise<ProductList>` — получает каталог товаров с сервера;
    
-   `createOrder(orderData): Promise<OrderSuccess>` — отправляет данные заказа на сервер.
    

### Компоненты интерфейса (`components/view/`)

#### 1. `Header` — шапка сайта

Отображается на всех страницах.

-   отображает счётчик товаров в корзине (`counter`).
    

#### 2. `Gallery` — галерея каталога

Отображает список товаров.

-   свойство `catalog: HTMLElement[]` — список карточек товаров;
    
-   отвечает за рендер сетки товаров.
    

#### 3. `Modal` — модальное окно

Универсальный компонент для всплывающих окон.

-   методы:
    
    -   `render({ content })` — отображает контент внутри модального окна;
        
    -   `open()` — открывает модальное окно;
        
    -   `close()` — закрывает модальное окно.
        

#### 4. `CardCatalog` — карточка товара в каталоге

Отображает товар в галерее.

-   поля: `title`, `price`, `category`, `image` — данные товара;
    
-   событие: `onClick` — клик по карточке (открывает детальный просмотр).
    

#### 5. `CardPreview` — превью товара в модальном окне

Детальный просмотр товара.

-   поля: `title`, `price`, `description`, `category`, `image`, `buttonText`, `buttonLocked` — полная информация о товаре;
    
-   событие: `onButtonClick` — действие кнопки («В корзину»/«Удалить»).
    

#### 6. `BaseBasket` — интерфейс корзины

Отображает содержимое корзины.

-   свойства: `items: HTMLElement[]`, `total: number` — список товаров и общая сумма;
    
-   отображает «Корзина пуста», если `items.length === 0`.
    

#### 7. `OrderForm` и `ContactsForm` — формы оформления заказа

Собирают данные для оформления заказа.

-   поля ввода: `payment`, `address`, `email`, `phone` — данные заказа;
    
-   флаг `valid` — валидность формы (можно ли отправить);
    
-   `errorMessages` — сообщения об ошибках валидации.
    

#### 8. `Success` — экран успеха

Показывает результат успешного заказа.

-   поле `total: number` — сумма заказа;
    
-   отображается после успешной отправки заказа.
    

## Поток данных и обработка событий

1.  **Загрузка данных**:
    
    -   `WebLarekApi.getProducts()` → `Products.setItems()` → событие `'products:loaded'`.
        
2.  **Отображение каталога**:
    
    -   обработчик `'products:loaded'` → создаёт `CardCatalog` для каждого товара → `Gallery.catalog = productCards`.
        
3.  **Выбор товара**:
    
    -   клик по `CardCatalog` → событие `'product:selected'` → `Products.setSelectedItem()`.
        
4.  **Просмотр деталей**:
    
    -   обработчик `'product:selected'` → рендер `CardPreview` → `Modal.render()`.
        
5.  **Управление корзиной**:
    
    -   клик по кнопке в `CardPreview` → событие `'product:action:trigger'` → `Basket.addItem()`/`removeItem()` → событие `'cart:updated'`.
        
6.  **Оформление заказа**:
    
    -   клик по кнопке корзины → событие `'cart:open'` → рендер `BaseBasket` в `Modal`;
        
    -   клик «Оформить заказ» → событие `'checkout:initiate'` → рендер `OrderForm` в `Modal`;
        
    -   ввод данных → обновление `Buyer` → валидация → событие `'order:updated'` → рендер форм;
        
    -   клик «Оплатить» → событие `'contact:submit'` → `WebLarekApi.createOrder()` → экран успеха.
        
7.  **Успешный заказ**:
    
    -   ответ сервера → `Success.total = result.total` → `Modal.render()`;
        
    -   очистка `Basket` и `Buyer` → закрытие модального окна.
        

## Ключевые события (`EventEmitter`)

-   `'products:loaded'` — каталог загружен с сервера;
    
-   `'product:selected'` — пользователь выбрал товар для просмотра;
    
-   `'cart:updated'` — корзина изменена (товар добавлен/удален);
    
-   `'cart:open'` — пользователь открыл корзину;
    
-   `'checkout:initiate'` — начат процесс оформления заказа;
    
-   `'product:action:trigger'` — действие с товаром (добавить/удалить);
