# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения (MVP \+ событийная модель)

Приложение построено по паттерну **MVP (Model‑View‑Presenter)** с использованием **событийно‑ориентированного подхода** через `EventEmitter`.

### Model (`Products`, `Basket`, `Buyer`, `WebLarekApi`)

Отвечает за данные и бизнес‑логику:

-   хранит и управляет данными;
    
-   реализует бизнес‑логику (добавление в корзину, валидация данных);
    
-   взаимодействует с API.
    

### View (`Header`, `Gallery`, `Modal`, `CardCatalog` и др.)

Отвечает за отображение:

-   отображает данные в DOM;
    
-   не содержит бизнес‑логики;
    
-   эмиттит события при взаимодействии пользователя (клик, ввод).
    

### Presenter (код в `main.ts`)

Координирующий компонент:

-   координирует взаимодействие Model и View;
    
-   подписывается на события от View и моделей;
    
-   вызывает методы моделей для изменения данных;
    
-   обновляет View при изменении данных.
    

## Ключевые классы и интерфейсы

### Базовый код (`components/base/`)

#### 1. `Component<T>` — базовый класс для всех UI‑компонентов

Базовый строительный блок всех визуальных компонентов.

-   конструктор: `constructor(container: HTMLElement)` — принимает контейнер для рендера;
    
-   поле: `container: HTMLElement` — ссылка на DOM‑элемент‑контейнер;
    
-   метод: `render(data?: Partial<T>): HTMLElement` — рендерит компонент с данными.
    

#### 2. `Api` — базовый класс для HTTP‑запросов

Обеспечивает взаимодействие с сервером.

-   конструктор: `constructor(baseUrl: string, options: RequestInit = {})` — инициализирует API с базовым URL;
    
-   методы: `get(uri: string)` — GET‑запрос, `post(uri: string, data: object)` — POST‑запрос.
    

#### 3. `EventEmitter` — брокер событий (паттерн «Наблюдатель»)

Механизм коммуникации между компонентами.

-   поля: `_events: Map<string | RegExp, Set<Function>>` — хранилище обработчиков событий;
    
-   методы:
    
    -   `on(event, callback)` — подписка на событие;
        
    -   `emit(event, data)` — генерация события с данными;
        
    -   `trigger(event, context)` — вызов обработчиков события.
        

### Модели данных (`components/models/`)

#### 1. `Products` — каталог товаров

Управляет данными о товарах.

-   `setItems(items: IProduct[])` — загружает товары в каталог;
    
-   `getItems(): IProduct[]` — возвращает все товары;
    
-   `getItem(id: string): IProduct` — ищет товар по ID (выбрасывает ошибку, если не найден);
    
-   `findItem(id: string): IProduct | null` — ищет товар, возвращает `null`, если не найден;
    
-   `setSelectedItem(id: string)` — устанавливает выбранный товар для детального просмотра;
    
-   `getSelectedItem(): IProduct | null` — возвращает выбранный товар.
    

#### 2. `Basket` — корзина покупок

Управляет корзиной пользователя.

-   `addItem(item: IProduct)` — добавляет товар в корзину;
    
-   `removeItem(id: string)` — удаляет товар по ID;
    
-   `clear()` — очищает корзину;
    
-   `getTotal(): number` — возвращает общую стоимость товаров в корзине;
    
-   `getCount(): number` — возвращает количество товаров в корзине.
    

#### 3. `Buyer` — данные покупателя

Хранит и валидирует данные заказчика.

-   геттеры/сеттеры для `payment`, `address`, `email`, `phone` — управление данными формы;
    
-   `validate(): BuyerErrors` — валидирует данные, возвращает объект с ошибками;
    
-   `clear()` — очищает данные после успешного заказа.
    

#### 4. `WebLarekApi` — слой коммуникации с сервером

Конкретная реализация API для проекта.

-   `getProducts(): Promise<ProductList>` — получает каталог товаров с сервера;
    
-   `createOrder(orderData): Promise<OrderSuccess>` — отправляет данные заказа на сервер.
    

### Компоненты интерфейса (`components/view/`)

#### 1. `Header` — шапка сайта

Отображается на всех страницах.

-   отображает счётчик товаров в корзине (`counter`).
    

#### 2. `Gallery` — галерея каталога

Отображает список товаров.

-   свойство `catalog: HTMLElement[]` — список карточек товаров;
    
-   отвечает за рендер сетки товаров.
    

#### 3. `Modal` — модальное окно

#### Назначение

Класс `Modal` — это компонент для управления модальными окнами в веб‑приложении. Он позволяет:

-   открывать и закрывать модальные окна;
    
-   передавать в них контент;
    
-   отслеживать состояние окна (открыто/закрыто);
    
-   реагировать на внешние события.
    

#### Свойства

-   `closeButton` (`HTMLButtonElement`) — кнопка закрытия модального окна (ищется по селектору `.modal__close`).
    
-   `contentElement` (`HTMLElement`) — контейнер для отображения контента модального окна (ищется по селектору `.modal__content`).
    
-   `isOpen` (`boolean`) — флаг состояния окна:
    
    -   `true` — окно открыто;
        
    -   `false` — окно закрыто (начальное значение).
        

#### Методы

1.  **`isModalOpen(): boolean`**
    
    -   **Возвращает:** текущее состояние окна (`true` — открыто, `false` — закрыто).
        
2.  **`set content(value: HTMLElement | null)`**
    
    -   **Назначение:** установка или очистка контента внутри модального окна.
        
    -   **Параметр:** `value` — HTML‑элемент для отображения или `null` для очистки контента.
        
    -   **Логика работы:**
        
        -   если `value` передан, заменяет содержимое `contentElement` на переданный элемент;
            
        -   если `value` равен `null`, очищает содержимое `contentElement`.
            
3.  **`close(): void`**
    
    -   **Назначение:** закрытие модального окна.
        
    -   **Выполняемые действия:**
        
        1.  Проверяет, открыто ли окно (`isOpen === true`). Если нет — завершает выполнение.
            
        2.  Устанавливает `isOpen = false`.
            
        3.  Удаляет CSS‑класс `modal_active` у контейнера (скрывает окно).
            
        4.  Очищает контент через `this.content = null`.
            
        5.  Отправляет событие `'modal:close'` через `events.emit()`.
            
4.  **`open(data: IModal): void`**
    
    -   **Назначение:** открытие модального окна и отображение переданного контента.
        
    -   **Параметр:** `data` — объект с полем `content` типа `HTMLElement | null`.
        
    -   **Выполняемые действия:**
        
        1.  Устанавливает `isOpen = true`.
            
        2.  Добавляет CSS‑класс `modal_active` к контейнеру (показывает окно).
            
        3.  Вызывает метод `render(data)` для отрисовки переданных данных.
            
        4.  Отправляет событие `'modal:open'` через `events.emit()`.
            

#### События

Класс генерирует следующие события через объект `events`:

-   `'modal:open'` — отправляется сразу после открытия модального окна;
    
-   `'modal:close'` — отправляется сразу после закрытия модального окна.
        

#### 4. `CardCatalog` — карточка товара в каталоге

#### Назначение

Класс `CardCatalog` — это компонент карточки товара/элемента каталога в веб‑приложении. Он расширяет базовый класс `BaseCard` и добавляет функциональность для отображения:

-   категории товара с визуальной стилизацией;
    
-   изображения товара из CDN;
    
-   обработки кликов по карточке.
    

#### Свойства

-   `categoryElement` (`HTMLElement`) — элемент для отображения категории товара (ищется по селектору `.card__category`).
    
-   `imageElement` (`HTMLImageElement`) — элемент изображения товара (ищется по селектору `.card__image`).
    

#### Конструктор

typescript

```typescript
constructor(events: IEvents, container: HTMLElement, actions?: ICardCatalog)
```

**Параметры:**

-   `events` (`IEvents`) — объект для работы с событиями приложения;
    
-   `container` (`HTMLElement`) — HTML‑элемент, представляющий собой контейнер карточки;
    
-   `actions?` (`ICardCatalog`) — опциональный объект с действиями, содержит обработчик клика `onClick`.
    

**Действия в конструкторе:**

1.  Инициализация базовых элементов через функцию `ensureElement`:
    
    -   элемента категории;
        
    -   элемента изображения.
        
2.  Если передан обработчик `onClick` в `actions`, устанавливается обработчик клика на контейнере карточки.
    

#### Методы

1.  **`set category(value: string)`**
    
    -   **Назначение:** установка и визуальное оформление категории товара.
        
    -   **Параметр:** `value` — название категории в виде строки.
        
    -   **Логика работы:**
        
        -   устанавливает текст категории в `categoryElement`;
            
        -   удаляет все CSS‑классы стилизации категорий из `categoryElement`;
            
        -   получает соответствующий CSS‑класс для категории из `categoryMap`;
            
        -   если класс найден, добавляет его к `categoryElement` для визуальной стилизации.
            
2.  **`set image(value: string)`**
    
    -   **Назначение:** установка изображения товара.
        
    -   **Параметр:** `value` — путь к изображению относительно CDN.
        
    -   **Логика работы:**
        
        -   формирует полный URL изображения, объединяя `CDN_URL` и переданный путь;
            
        -   вызывает метод `setImage()` для установки изображения в `imageElement`.
            

#### Интерфейсы

-   `ICardCatalog` — интерфейс действий для карточки каталога:
    
    typescript
    
    ```typescript
    interface ICardCatalog {
        onClick?: () => void;
    }
    ```
    
    Содержит опциональное свойство `onClick` — обработчик клика по карточке.
    

#### Зависимости и константы

Класс использует следующие внешние зависимости:

-   `BaseCard`, `ICard` — базовый класс и интерфейс карточки;
    
-   `IEvents` — интерфейс событий приложения;
    
-   `ensureElement` — утилита для поиска элементов в DOM;
    
-   `CDN_URL` — базовый URL CDN для загрузки изображений;
    
-   `categoryMap` — объект‑маппинг названий категорий на CSS‑классы для стилизации.
    

#### События

Класс не генерирует собственных событий напрямую, но может реагировать на клик через переданный обработчик `onClick`.


    

#### 5. `CardPreview` — превью товара в модальном окне

#### Назначение

Класс `CardPreview` — компонент детальной карточки товара (предпросмотра) в веб‑приложении. Расширяет базовый класс `BaseCard` и предоставляет функционал для:

-   отображения полной информации о товаре (категория, описание, изображение, кнопка действия);
    
-   стилизации категории товара по типу;
    
-   управления состоянием кнопки (активна/заблокирована);
    
-   обработки клика по кнопке действия.
    

#### Свойства

-   `categoryElement` (`HTMLElement`) — элемент для отображения категории товара (ищется по селектору `.card__category`).
    
-   `textElement` (`HTMLElement`) — элемент для текста описания товара (ищется по селектору `.card__text`).
    
-   `buttonElement` (`HTMLButtonElement`) — кнопка действия на карточке (ищется по селектору `.card__button`).
    
-   `imageElement` (`HTMLImageElement`) — элемент изображения товара (ищется по селектору `.card__image`).
    

#### Методы

1.  **`set category(value: keyof typeof categoryMap)`**
    
    -   **Назначение:** установка и визуальное оформление категории товара.
        
    -   **Параметр:** `value` — название категории (ключ из `categoryMap`).
        
    -   **Логика работы:**
        
        -   устанавливает текст категории в `categoryElement`;
            
        -   сбрасывает классы `categoryElement` до базового (`'card__category'`);
            
        -   получает соответствующий CSS‑класс для категории из `categoryMap`;
            
        -   если класс найден, добавляет его к `categoryElement` для стилизации.
            
2.  **`set description(value: string)`**
    
    -   **Назначение:** установка описания товара.
        
    -   **Параметр:** `value` — текст описания.
        
    -   **Логика работы:** устанавливает текст в `textElement`.
        
3.  **`set buttonText(value: string)`**
    
    -   **Назначение:** установка текста на кнопке действия.
        
    -   **Параметр:** `value` — текст для кнопки.
        
    -   **Логика работы:** устанавливает текст в `buttonElement`.
        
4.  **`set buttonLocked(value: boolean)`**
    
    -   **Назначение:** управление состоянием кнопки (активна/заблокирована).
        
    -   **Параметр:** `value` — `true` для блокировки, `false` для активации.
        
    -   **Логика работы:** устанавливает свойство `disabled` у `buttonElement` в соответствии с `value`.
        
5.  **`set image(value: string)`**
    
    -   **Назначение:** установка изображения товара.
        
    -   **Параметр:** `value` — путь к изображению относительно CDN.
        
    -   **Логика работы:**
        
        -   формирует полный URL изображения, объединяя `CDN_URL` и переданный путь;
            
        -   вызывает метод `setImage()` для установки изображения в `imageElement`.
            

#### Зависимости и константы

Класс использует следующие внешние зависимости:

-   `BaseCard`, `ICard` — базовый класс и интерфейс карточки;
    
-   `IEvents` — интерфейс событий приложения;
    
-   `ensureElement` — утилита для поиска элементов в DOM;
    
-   `CDN_URL` — базовый URL CDN для загрузки изображений;
    
-   `categoryMap` — объект‑маппинг названий категорий на CSS‑классы для стилизации.

#### 6. `BaseBasket` — интерфейс корзины

#### Назначение

Класс `BaseBasket` — базовый компонент корзины в веб‑приложении. Расширяет класс `Component` и предоставляет функционал для:

-   отображения списка товаров в корзине;
    
-   обновления общей стоимости заказа;
    
-   управления состоянием кнопки оформления заказа (активна/неактивна);
    
-   отправки события при нажатии на кнопку оформления заказа.
    

#### Свойства

-   `listElement` (`HTMLElement`) — контейнер для списка товаров (ищется по селектору `.basket__list`).
    
-   `totalElement` (`HTMLElement`) — элемент для отображения общей стоимости (ищется по селектору `.basket__price`).
    
-   `submitButton` (`HTMLButtonElement`) — кнопка оформления заказа (ищется по селектору `.basket__button`).
    

#### Методы

1.  **`set items(items: HTMLElement[])`**
    
    -   **Назначение:** обновление списка товаров в корзине.
        
    -   **Параметр:** `items` — массив HTML‑элементов карточек товаров.
        
    -   **Логика работы:**
        
        -   заменяет содержимое `listElement` на переданный массив элементов с помощью `replaceChildren(...items)`;
            
        -   отключает `submitButton`, если корзина пуста (`items.length === 0`), иначе включает её.
            
2.  **`set total(value: number)`**
    
    -   **Назначение:** установка общей стоимости заказа.
        
    -   **Параметр:** `value` — числовое значение общей стоимости.
        
    -   **Логика работы:** устанавливает текст в `totalElement` в формате `${value} синапсов` (например, «1500 синапсов»).
        

#### События

Класс генерирует следующее событие через объект `events`:

-   `'basket:submit'` — отправляется при клике на кнопку оформления заказа (`submitButton`). Это событие можно перехватить для запуска процесса оформления заказа (перехода к форме оплаты и т. д.).
    


    

#### 7. `OrderForm` и `ContactsForm` — формы оформления заказа

#### Общее назначение

Классы `OrderForm` и `ContactsForm` — компоненты последовательных шагов оформления заказа в веб‑приложении. Они расширяют класс `BaseForm` и обеспечивают:

-   сбор данных от пользователя;
    
-   отслеживание изменений в реальном времени;
    
-   отправку событий для координации с другими компонентами;
    
-   возможность программного управления содержимым полей.
    

* * *

### Класс `OrderForm`

**Назначение:** форма выбора способа оплаты и ввода адреса доставки.

#### Свойства

-   `buttonCard` (`HTMLButtonElement`) — кнопка выбора оплаты картой (селектор: `button[name="card"]`).
    
-   `buttonCash` (`HTMLButtonElement`) — кнопка выбора оплаты при получении (селектор: `button[name="cash"]`).
    
-   `addressInput` (`HTMLInputElement`) — поле ввода адреса доставки (селектор: `input[name="address"]`).
    

#### Методы

1.  **`set payment(value: TPayment | null)`**
    
    -   **Назначение:** установка и визуальное выделение выбранного способа оплаты.
        
    -   **Параметр:** `value` — способ оплаты (`'card'` или `'cash'`) либо `null`.
        
    -   **Логика работы:** переключает класс `button_alt-active` на соответствующих кнопках.
        
2.  **`set address(value: string)`**
    
    -   **Назначение:** установка значения в поле адреса.
        
    -   **Параметр:** `value` — текст адреса доставки.
        
    -   **Логика работы:** устанавливает значение в `addressInput`.
        

#### События

-   `'order.payment:change'` — при выборе способа оплаты, передаёт `{ payment: 'card' }` или `{ payment: 'cash' }`.
    
-   `'order.address:change'` — при вводе в поле адреса, передаёт `{ address: <текущее_значение> }`.
    

* * *

### Класс `ContactsForm`

**Назначение:** форма ввода контактных данных (email и телефон) для завершения оформления заказа.

#### Свойства

-   `emailInput` (`HTMLInputElement`) — поле ввода email (селектор: `input[name="email"]`).
    
-   `phoneInput` (`HTMLInputElement`) — поле ввода телефона (селектор: `input[name="phone"]`).
    
-   `errorContainer` (`HTMLElement | null`) — контейнер для отображения ошибок валидации (создаётся динамически, класс `form-errors`).
    

#### Методы

1.  **`set email(value: string)`**
    
    -   **Назначение:** установка значения в поле email.
        
    -   **Параметр:** `value` — текст email.
        
    -   **Логика работы:** устанавливает значение в `emailInput`.
        
2.  **`set phone(value: string)`**
    
    -   **Назначение:** установка значения в поле телефона.
        
    -   **Параметр:** `value` — текст номера телефона.
        
    -   **Логика работы:** устанавливает значение в `phoneInput`.
        

#### События

-   `'contacts.email:change'` — при вводе в поле email, передаёт `{ email: <текущее_значение> }`.
    
-   `'contacts.phone:change'` — при вводе в поле телефона, передаёт `{ phone: <текущее_значение> }`.

#### 8. `Success` — экран успеха

#### Назначение

Класс `Success` — компонент экрана успешного оформления заказа в веб‑приложении. Расширяет класс `Component` и предназначен для:

-   отображения итоговой суммы списания («Списано X синапсов»);
    
-   предоставления кнопки для закрытия экрана успеха;
    
-   отправки события при закрытии экрана.
    

#### Свойства

-   `descriptionElement` (`HTMLElement`) — элемент для отображения текста с суммой списания (ищется по селектору `.order-success__description`).
    
-   `closeButton` (`HTMLButtonElement`) — кнопка для закрытия экрана успеха (ищется по селектору `.order-success__close`).
    

#### Методы

1.  **`set total(value: number)`**
    
    -   **Назначение:** установка и отображение суммы списания.
        
    -   **Параметр:** `value` — числовое значение общей стоимости заказа.
        
    -   **Логика работы:** устанавливает текст в `descriptionElement` в формате `Списано ${value} синапсов` (например, «Списано 1500 синапсов»).
        

#### События

Класс генерирует следующее событие через объект `events`:

-   `'success:close'` — отправляется при клике на кнопку закрытия экрана (`closeButton`). Это событие можно перехватить для:
    
    -   закрытия модального окна с экраном успеха;
        
    -   сброса состояния корзины;
        
    -   возврата пользователя в основной каталог товаров.
    

## Поток данных и обработка событий

1.  **Загрузка данных**:
    
    -   `WebLarekApi.getProducts()` → `Products.setItems()` → событие `'products:loaded'`.
        
2.  **Отображение каталога**:
    
    -   обработчик `'products:loaded'` → создаёт `CardCatalog` для каждого товара → `Gallery.catalog = productCards`.
        
3.  **Выбор товара**:
    
    -   клик по `CardCatalog` → событие `'product:selected'` → `Products.setSelectedItem()`.
        
4.  **Просмотр деталей**:
    
    -   обработчик `'product:selected'` → рендер `CardPreview` → `Modal.render()`.
        
5.  **Управление корзиной**:
    
    -   клик по кнопке в `CardPreview` → событие `'product:action:trigger'` → `Basket.addItem()`/`removeItem()` → событие `'cart:updated'`.
        
6.  **Оформление заказа**:
    
    -   клик по кнопке корзины → событие `'cart:open'` → рендер `BaseBasket` в `Modal`;
        
    -   клик «Оформить заказ» → событие `'checkout:initiate'` → рендер `OrderForm` в `Modal`;
        
    -   ввод данных → обновление `Buyer` → валидация → событие `'order:updated'` → рендер форм;
        
    -   клик «Оплатить» → событие `'contact:submit'` → `WebLarekApi.createOrder()` → экран успеха.
        
7.  **Успешный заказ**:
    
    -   ответ сервера → `Success.total = result.total` → `Modal.render()`;
        
    -   очистка `Basket` и `Buyer` → закрытие модального окна.
        

## Ключевые события (`EventEmitter`)

-   `'products:loaded'` — каталог загружен с сервера;
    
-   `'product:selected'` — пользователь выбрал товар для просмотра;
    
-   `'cart:updated'` — корзина изменена (товар добавлен/удален);
    
-   `'cart:open'` — пользователь открыл корзину;
    
-   `'checkout:initiate'` — начат процесс оформления заказа;
    
-   `'product:action:trigger'` — действие с товаром (добавить/удалить);
